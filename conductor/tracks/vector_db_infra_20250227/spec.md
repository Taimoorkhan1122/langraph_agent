# Track: Vector Database Infrastructure

## Objective

Establish the foundational vector database layer for the LangGraph agent system: run Weaviate in Docker, define a multi-tenant schema for document Q&A, and insert sample data so the RAG agent can be built and tested.

## Scope

- **In scope:** Docker Compose for Weaviate, multi-tenant Document collection schema (fileId, question, answer, pageNumber), creation of one tenant and insertion of at least three sample document entries, health checks and basic documentation.
- **Out of scope:** LangGraph/LangChain integration, RAG agent implementation, Chart tool, streaming APIs.

## Requirements Summary

| ID | Requirement | Source |
|----|-------------|--------|
| R1 | docker-compose.yml with Weaviate service (image, ports, env, volumes) | US-001 |
| R2 | Weaviate starts via `docker-compose up`, healthy at localhost:8080, REST 200 on health | US-001 |
| R3 | Document collection with multiTenancyConfig.enabled: true | US-002 |
| R4 | Schema fields: fileId (string, no vector/index), question (text, vectorized), answer (text, vectorized), pageNumber (text[]) | US-002 |
| R5 | Schema created via Weaviate JS client with error handling | US-002 |
| R6 | One tenant created; ≥3 document entries inserted with fileId, question, answer, pageNumber | US-003 |
| R7 | Vectors auto-generated by Weaviate; data verifiable via query | US-003 |

## Acceptance Criteria (from Project User Stories)

- Docker Compose file exists; container runs and is healthy; Weaviate REST API responds 200 at health endpoint; volumes and env vars documented.
- GET /v1/schema returns Document class with multi-tenancy and correct field types/indexing.
- At least three sample entries retrievable; each has required fields; vectors present.

## Dependencies

- Node.js, pnpm, Docker and Docker Compose.
- Weaviate JS client (to be added to package.json in this track if not present).

## Definition of Done

- All acceptance criteria above met.
- Unit or integration tests for schema creation and data insertion where applicable.
- README or docs updated with run instructions and env vars.
